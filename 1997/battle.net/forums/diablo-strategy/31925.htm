I am running out the door to a meeting, so I only have time to jot 
down a few quick thoughts before this thread disappears on page 2.

First, I think that some of the more difficult issues about low level 
characters being helped by higher level characters, etc., probably 
are artifacts of the way the game was created.  As I understand it, 
Diablo was initially conceived as a single player game and 
multiplayer features were added later, almost as an afterthought.  In 
solo play, the character can't get items from someone else, can't 
have someone else buy elixirs or other items for them, can't use 
someone else's portal to shortcut to Hell, etc.  These only become 
possible (and difficult issues) due to the multiplayer feature.  
These issues may well have resolved as part of the game design if 
Diablo had started out with multiplayer play as a possibility.  

Second, I agree with you, CM, that many of these actions make the 
game less challenging, lessen risk, appear to be contrary to 
"role playing" the game, etc.  But I can't call it 
"cheating."  When we run into real problems with applying our 
definition of cheating, we need to re-examine basic assumptions, such 
as our definition of cheating. Or if our definition of cheating seems 
to be workable, perhaps what we really need is a new term or word to 
describe this type of action (e.g., "unsporting")?

Third, with regard to the use of backup programs, Thecla, do you see 
any problem with backing up the character information files as 
instructed by Blizzard?  If not, how is that different from a program 
like Kevin's that is run while Diablo is not running?  Both use 
"external" programs: Kevin's program and the DOS/Windows 
operating system or equivalent (you use the operating system when you 
type the "copy" command).  The internal/external program may 
seem like a clean line to draw, but it really isn't, as evidenced by 
bugs (internal) and Blizzard's sanctioning of copying the character 
files.

While it's a tough call, since Blizzard appears to have no problem 
with people copying their data files for certain purposes, I can't 
call this cheating.  By analogy to my old roleplaying days with AD&D, 
all my character "data" is stored on a sheet of paper.  If I 
lost that paper, or it was destroyed or rendered illegible for some 
reason, would I be cheating to fill out a new character data sheet 
for the next session. Or, if I were foresightful enough, use a copy 
of the character data sheet?

I do not view the Diablo character data files as my character any 
more than I viewed my data sheet as my AD&D character.  My character 
is a fusion of that data, the game design, and your imagination (and 
the imagination of the other players).  Can someone who clones my 
character data really be said to be playing my character, even though 
all the character data is identical?  No.  

Hmmm . . . I didn't mean to get so philosophical.  Must be the time 
of day!
