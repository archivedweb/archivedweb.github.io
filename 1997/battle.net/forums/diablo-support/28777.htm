Your problem is a conflict between your dos and windows cd drivers. I 
had the same trouble and found out when I used incompatible dos cd 
drivers, but then eventhing worked fine (except in native DOS because 
i didn't have any CD drivers until I unREMed the correct drivers).
just go to the dos prompt
edit autoexec.bat
use rem in front of the cd driver if one is there
save it
open file config.sys, use the rem statement again (this allows you to 
keep the drivers for DOS native programs.  Your alternative is to 
delete the line and you never want to do that) in front of the line.  
Then save it.
Reboot the computer and you should be able to run Diablo.  
If you can't, you might need a windows cd driver.  Go to add new 
hardware and follow the promts.  If nothing new came up, you have the 
cd driver already installed.
